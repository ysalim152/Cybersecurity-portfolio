# 📌 Project Overview

This project demonstrates how to analyze and respond to a cybersecurity incident using the **NIST Cybersecurity Framework (CSF)**.

The scenario is based on a simulated **Denial of Service (DoS)** attack targeting a multimedia company's internal network. The attack was caused by a flood of ICMP packets that exploited a firewall configuration weakness.

The objective of this project is to investigate the incident, identify the root cause, assess the business impact, and recommend security improvements following the five NIST CSF functions.

# 🏢 Business Scenario

A multimedia company providing web design and digital marketing services experienced a **Denial-of-Service (DoS)** attack.

An attacker exploited an improperly configured firewall and flooded the network with ICMP packets.

As a result:

- Internal services became unavailable
- Employees could not access network resources
- Network availability was interrupted for approximately two hours

The cybersecurity team successfully contained the attack and initiated a post-incident investigation.

# 🧭 NIST Cybersecurity Framework

The incident was analyzed following the five NIST CSF functions.

## 1️⃣ Identify

- Identify critical assets
- Identify affected systems
- Analyze vulnerabilities
- Determine business impact

---

## 2️⃣ Protect

Implemented controls:

- Firewall hardening
- ICMP Rate Limiting
- Source IP Verification
- Security Policies
- User Awareness

---

## 3️⃣ Detect

Detection improvements:

- Intrusion Detection System (IDS)
- Intrusion Prevention System (IPS)
- Network Monitoring
- Log Analysis
- Traffic Baselines

---

## 4️⃣ Respond

Response actions included:

- Blocking malicious ICMP traffic
- Isolating affected systems
- Collecting logs
- Performing root cause analysis
- Updating firewall rules

---

## 5️⃣ Recover

Recovery activities:

- Restoring critical services
- Validating firewall configuration
- Monitoring the network
- Documenting lessons learned
